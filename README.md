                                                                                 VERSION CONTROL AND REPOSITORY MANAGEMENT USING GIT AND GITHUB

AIM:
To understand and implement version control using Git and GitHub by creating a repository, adding files, making commits, and managing version history. This lab helps in learning how to collaborate on projects, track changes, and maintain code efficiently using GitHub.

PROCEDURE:
Step 1: Create a GitHub Account
1.	Go to GitHub.
2.	Click Sign Up in the top-right corner.
3.	Enter a username, email, and password and complete the verification.
4.	Click Create account and verify your email.

Step 2: Create a New Repository
1.	After logging in, click the "+" icon in the top-right corner.
2.	Select "New repository".
3.	Enter a repository name (e.g., MyFirstRepo).
4.	Choose "Public" or "Private".
5.	Check the box "Add a README file" (this initializes the repo with a README).
6.	Click Create repository.
 
Step 3: Upload a File to GitHub
1.	Open your newly created repository.
2.	Click "Add file" → "Upload files".
3.	Click "Choose your files" and select the file you want to upload.
4.	Add a commit message (e.g., "Added first file").
5.	Click "Commit changes".

Result:
Successfully created a GitHub repository, initialized it with a README file, uploaded files, and managed version control using Git.
